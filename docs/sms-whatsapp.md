# SMS/WhatsApp Agent POC

This guide sets up the Gradient Bang SMS agent with Twilio WhatsApp Sandbox for
local testing.

## 1. Start Local Services

Start the Supabase edge functions so the SMS agent can call
`sms_agent_tools`:

```bash
uv run functions
```

In another terminal, start the SMS agent server:

Create `.env.sms` in the repo root:

```bash
SMS_AGENT_CHARACTER_ID=<your-character-uuid>
SUPABASE_URL=<your-supabase-url>
SUPABASE_ANON_KEY=<your-anon-key>
EDGE_API_TOKEN=<same-edge-token-if-required>
GOOGLE_API_KEY=<your-google-key>

# Optional
SMS_AGENT_HOST=0.0.0.0
SMS_AGENT_PORT=8008
SMS_AGENT_LLM_PROVIDER=google
SMS_AGENT_LLM_MODEL=gemini-2.5-flash
SMS_AGENT_PERSONALITY_TONE="Decommissioned Federation military AI. Formal, slightly archaic phrasing. References standard protocol and regulation. Addresses the player as commander."
SMS_AGENT_MAP_PLACEHOLDER_IMAGE_URL=https://api.gradient-bang.com/storage/v1/object/public/GB%20Public/map-test.png
```

The SMS agent loads `.env.bot` first, then `.env.sms` with SMS-specific values
taking precedence.

Start the server:

```bash
uv run --group sms-agent sms-agent
```

By default, the server listens on `http://localhost:8008`.

Check it:

```bash
curl http://localhost:8008/health
```

## 2. Expose the Server

Twilio needs a public HTTPS URL. Use ngrok for local testing:

```bash
ngrok http 8008
```

Copy the HTTPS forwarding URL, for example:

```text
https://example.ngrok-free.app
```

## 3. Activate Twilio WhatsApp Sandbox

1. Open the Twilio Console.
2. Go to **Messaging > Try it out > Send a WhatsApp message**.
3. Follow the Sandbox instructions.
4. From your phone, send the displayed `join <sandbox-code>` message to the
   Twilio Sandbox WhatsApp number.
5. Wait for Twilio to confirm that your WhatsApp account joined the Sandbox.

## 4. Configure the Inbound Webhook

In Twilio Console:

1. Open the WhatsApp Sandbox settings.
2. Find **When a message comes in**.
3. Set the URL to your ngrok URL with `/whatsapp` appended:

```text
https://example.ngrok-free.app/whatsapp
```

4. Set the method to `POST`.
5. Save.

## 5. Greeting Behavior

If a user sends a greeting such as `hi`, `hello`, or `start`, the agent runs
normal LLM inference with the configured character identity. It should answer in
the ship-AI voice rather than listing tools.

Non-tool questions also go through LLM inference. Tool-shaped requests still run
their backing tool first, then the tool result is passed to the LLM for the final
WhatsApp reply.

Button replies from Twilio are still handled through `ButtonText` /
`ButtonPayload` when present. The supported payloads are `status`, `ships`,
`corporation_info`, and `map`.

## 6. Test from WhatsApp

Send a message to the Twilio Sandbox number from your phone:

```text
status
```

Other supported POC commands:

```text
ships
corporation
map
```

`map` returns a normal SMS-agent text reply plus a placeholder image using
TwiML `<Media>`. The image URL must be public HTTPS. Override it with
`SMS_AGENT_MAP_PLACEHOLDER_IMAGE_URL` if you want to use your own hosted image.

Expected flow:

```text
WhatsApp -> Twilio Sandbox -> ngrok -> local FastAPI /whatsapp
  -> run_turn_response() -> sync tool -> LLM -> TwiML reply
```

You should receive a WhatsApp reply generated by the SMS agent.

To test greeting inference, send:

```text
hi
```

## Local Smoke Test

Before using Twilio, fake a Twilio webhook locally:

```bash
curl -X POST http://localhost:8008/whatsapp \
  -H "Content-Type: application/x-www-form-urlencoded" \
  --data-urlencode "Body=status" \
  --data-urlencode "From=whatsapp:+15555555555" \
  --data-urlencode "MessageSid=test"
```

The response should be TwiML XML:

```xml
<Response><Message>...</Message></Response>
```

## Troubleshooting

- `SMS_AGENT_CHARACTER_ID is not set`: export `SMS_AGENT_CHARACTER_ID` before
  starting `sms-agent`.
- Auth errors from `sms_agent_tools`: check `EDGE_API_TOKEN`, local auth bypass
  settings, or `SMS_AGENT_ACCESS_TOKEN` if you are using user auth.
- No request reaches the app: confirm ngrok is running and the Twilio webhook is
  exactly `https://<ngrok-host>/whatsapp`.
- LLM failures: confirm the provider API key is set, for example
  `GOOGLE_API_KEY`, or set `SMS_AGENT_LLM_PROVIDER` to match your available key.
- Ngrok URLs change when restarted unless you use a reserved domain. Update the
  Twilio Sandbox webhook after restarting ngrok.
